# synthetic test fixtures

Hand-authored or hand-trimmed fixtures, distinct from the captured
real traffic under `../mappings/` and `../__files/`. The synthetic
versions are deterministic and small — preferred for tests that don't
need real archive.org content.

## What's in here

### `data.zip` (synthetic catalog)

Slim subset of the real catalog — 6 hand-picked canonical shows
(1965 through 1993) plus the recordings they reference. Same schema
as the real `data.zip` (manifest, collections, shows/, recordings/),
so both platforms' data layers load it without code changes.

Loads in milliseconds vs. seconds for the full ~13MB real catalog,
which is the whole point.

**Regenerated by**:
```bash
uv run hermetic/scripts/build_synthetic_datazip.py
```

The script reads `../../dead-metadata/data.zip` (sibling directory)
as the source of truth and emits a slim copy. If you want to add or
swap a show, edit `CANONICAL_SHOWS` at the top of the script and
re-run.

## Future synthetic mappings

Hand-authored WireMock mappings (for scenarios where capturing real
traffic doesn't make sense — fault injection, error responses,
deliberately-broken edge cases) live alongside the data.zip in this
directory. None exist yet; add them as scenarios from
`docs/playback-test-scenarios.md` demand them.
